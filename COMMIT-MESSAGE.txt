fix: resolve critical crawl blockers and SEO issues (PR-1)

This commit fixes all critical issues identified in the October 16, 2025
crawl audit that were blocking Google and Bing from properly indexing
the site.

## Fixed Issues

### 1. Sitemap 400 Error (Critical)
- Added 1.5s timeout to Supabase fetch to prevent hanging
- Implemented Promise.race pattern for graceful fallback
- Sitemap now always returns 200 even if database is unavailable
- Falls back to static content (events, dungeons, core pages)

File: src/app/sitemap.ts

### 2. Footer Links 404 Errors (High Priority)
- Fixed /community-guidelines → /guidelines
- Changed /events/add → /contact?subject=Event%20Submission (temporary)
- Created placeholder pages for /accessibility and /report

Files:
- src/components/Footer.tsx
- src/app/accessibility/page.tsx (new)
- src/app/report/page.tsx (new)

### 3. Dungeons Page Duplication (Quality Issue)
- Added dedupeBySlug() utility function
- Implements Map-based deduplication by slug
- Applies stable sort by name for consistent rendering
- Eliminates duplicate dungeon entries

File: src/app/dungeons/DungeonsPageClient.tsx
